# Rediance — UIモック仕様書

SNS「Rediance」のUIプロトタイプです。プログラマがモックを見ながら仕様を理解するための資料です。

---

## プロジェクト概要

**コンセプト：「フォロワーの数だけが影響力じゃない」**

Redianceは、フォロワーをただカウントするのではなく、**誰にどう評価されているか**で影響力を測るSNSです。

フォロワーを5段階の階層（Circle）に分類し、上位Circleに入れてもらうほど影響力スコアが上がります。影響力の計算構造はGoogleのPageRankと同じ**再帰的な仕組み**です：

```
影響力スコア ＝ フォロワーの質（誰にフォローされているか）
             × Circle評価の深さ（どのCircleに入れられているか）
             ＋ 投稿のエンゲージメント
```

影響力の高い人にInnerに入れてもらうほど、自分のスコアも大きく上がります。

**数字を見せない思想：** つながり数・フォロワー数は非公開。見えるのは影響力スコアと称号だけ。数ではなく質で勝負するSNS。

## つながりの仕組み

Redianceには3つの関係レイヤーがあります。

### フォロー（一方的）

- 誰でも自由にフォローできる
- フォローした相手のストーリーズが見れる
- フォローしただけでもCircle階層に入れられる（影響力計算に一方的に貢献）
- **メッセージは送れない**

### コネクト（双方向）

コネクトが成立するとメッセージが送れるようになる。

**コネクトの成立方法：**

| 方法 | フロー |
|------|--------|
| QRコード | 対面でスキャン → 即コネクト成立 |
| ID検索 | ハンドルネームで検索 → 即コネクト成立 |
| 電話帳同期 | アドレス帳から知り合いを検出 → 即コネクト成立 |
| SNS上で発見 | フォロー → 相手に通知 → 相手が「コネクトする」を押す → 成立 |

**SNS上で発見した場合のフロー：**
1. Aさんが Bさんを「フォロー」する
2. Bさんに通知「Aさんがフォローしました」＋「コネクトする」ボタン
3. Bさんが「コネクトする」を押す
4. Aさんに通知「Bさんとコネクトしました」
5. **自動的に相互フォロー状態になる**
6. 双方がAmbient（最下層）に自動追加される
7. メッセージが送れるようになる

**コネクトしなかった場合：**
- フォロー関係は残る（Aさんは Bさんのストーリーズを見れる）
- AさんはBさんをCircle階層に入れられる（影響力計算には一方的に貢献する）
- メッセージは送れない

### Circle階層（個人の裏設定）

- フォローしている人を5段階に分類する**完全に個人的な設定**
- **相手にはどの階層に入れられているか見えない**
- フォローだけの関係でも、コネクト済みでも、どちらでも階層に入れられる

## 画面一覧と仕様

### 1. ホーム画面

ホームは「自分中心」の画面。以下の要素が上から順に並ぶ：

1. **ストーリーズアイコン列**（横スクロール） — フォローしている人のアバター。タップでストーリーズ再生
2. **自分のポケモンカード** — 称号・スコア・プログレスバー
3. **アクションボタン** — 💬メッセージ送信 / 🎉イベント作成 / 📦アーカイブ
4. **Circle階層メンバー一覧** — Inner〜Ambientまで全5階層。アバタータップでストーリーズ再生。多い階層は「もっと見る」で遷移
5. **Circle設定リンク** / **グループ管理リンク**

---

### 2. ストーリーズ

- フルスクリーン表示。インスタストーリーズと同じUX
- 上部にプログレスバー（複数投稿の進捗）
- 自動送り（5秒）。左タップで戻る、右タップで次へ
- 右上に✕ボタンで閉じる
- **左上のアバター/名前をタップ → 相手のプロフィールに遷移**
- コメント欄はなし。タグ付けされた人だけ「返事ストーリーズ」を作れる
- 投稿はAIが自動生成した背景デザイン付き（モックではCSSグラデーション）

**「✨ 強い光が届きました」：** フォローしていなくても、Sun級ユーザーの投稿がストーリーズに流れてくる。ゴールド背景で特別表示。

---

### 3. 投稿フロー — AIデザイン提案

1. テキストを入力
2. **「✨ アイデンティティに残す」チェック（デフォルトOFF）**
   - ONの場合 → カテゴリ選択（既存 or 新規作成）
   - ONの投稿はストーリーズ（24時間）＋アイデンティティ（永久保存）
   - OFFの投稿はストーリーズ（24時間）のみ。24時間後にアーカイブへ移動
3. 「AIデザインを生成」ボタンを押す
4. AIが4パターンの背景を提案（サイバー／ナチュラル／エレガント／ポップ）
5. ユーザーが好みのデザインを選んで投稿

---

### 4. 投稿のライフサイクル

```
投稿 → ストーリーズ（24時間表示）
         ↓ 24時間後
       アーカイブ（自分だけ見れる倉庫）
         ↓ 本人が選択
       アイデンティティに昇格（永久公開）
```

- **ストーリーズ** — 24時間で消える。全フォロワーに表示
- **アーカイブ** — 自分だけが見れる過去投稿の倉庫
- **アイデンティティ** — 投稿時に選択 or アーカイブから後で選択。プロフィールに永久表示

---

### 5. プロフィール — ポケモンカード風 + アイデンティティ

プロフィール画面の構成：
1. **ポケモンカード** — 称号・スコア・プログレスバー
2. **フォローボタン**（他人のプロフィールの場合）
3. **アイデンティティ** — 本人が残した投稿がカテゴリ別に並ぶ

#### ポケモンカード

称号（天体名）によってカードの外観が変わる：

| 称号 | ランク | カードデザイン |
|------|--------|--------------|
| ☀️ Sun（太陽） | 上位1% | ゴールド枠、キラキラアニメーション |
| 💫 Nova（新星） | 上位5% | 紫枠、紫グロー |
| 🌙 Moon（月） | 上位20% | シルバー枠、シルバーグロー |
| ⭐ Star（星） | 上位50% | ブルー枠 |
| 🕯️ Ember（残り火） | 下位50% | ダークブラウン |

**重要：** スコアはフォロワー数ではなく計算された影響力値。1人フォロワーが減っても大きく変動しない。称号は全ユーザー相対評価なのでインフレしない。

#### アイデンティティ

「自分はこういう人間です」を投稿で語る、自己紹介を超えた自己表現の場。

- 本人がカテゴリを自由に作成（例：「🇸🇬 シンガポール」「💼 経営」「🤖 AI・テック」）
- カテゴリは**最終更新順に自動ソート**される。新しい投稿があったカテゴリが上に上がる
- 訪問者は上から見れば「最近この人が力入れてるテーマ」がわかる
- カテゴリ内はインスタグリッド風（3列、背景グラデーションがサムネイル）
- タグ付けされた人のプロフィールにも飛べる → 人→投稿→人→投稿…と深掘りが止まらない

---

### 6. Circle（フォロワー階層）— 5段階

| Circle | 上限 | 意味 |
|--------|------|------|
| 🔥 Inner | 最大3人 | 最も信頼する存在 |
| 💎 Trusted | 最大10人 | 信頼する仲間 |
| ⭐ Close | 最大30人 | 近い存在 |
| 🤝 Connected | 最大100人 | 繋がり |
| 👥 Ambient | 制限なし | 一般フォロワー（デフォルト） |

**影響力への重み付け：**
- 他人にInnerに入れてもらう → 最も高いポイント
- Ambientに入れてもらう → ポイントなし
- フォロワー自身の影響力も重みに反映（Sunの人にInnerに入れられる ≫ Emberの人にInnerに入れられる）

**階層は完全非公開。** 相手にはどの階層に入れられているか見えない。

### 7. Circle設定画面

- 5階層を折りたたみ式で表示
- 「＋追加」ボタン → ボトムシートが開いてフォロワーを選択できる
- おすすめ表示＋検索バー付き

---

### 8. メッセージ機能

**一斉送信（Redianceの売り機能）：**
- Circle階層 or カスタムグループを宛先に選択
- 複数グループ/階層を同時に選択可能
- **受け取った人には個別DMとして届く。一斉送信だとは一切わからない**
- 送り手だけが全員の反応を俯瞰できる「ダッシュボード画面」を見れる

**一斉送信ダッシュボード（送り手のみ）：**
- 自分の送信メッセージが表示
- 各人の返信が時系列で並ぶ（アバター＋名前付き）
- 閲覧専用 — ここからは返信できない
- 個別に返信したい場合は、相手の名前をタップして1:1トーク画面に遷移

**1:1トーク：**
- 通常のチャットUI
- コネクト済みのユーザー同士のみ

**カスタムグループ：**
- ユーザーが自由に作れる私的なグループ分け（例：「酒好き」「投資仲間」）
- 相手からはグループの存在が見えない
- メッセージ/イベント招待の宛先として使える

---

### 9. イベント作成

- イベント名・日時・場所を入力
- 宛先をCircle階層 or カスタムグループから選択
- **招待は個別メッセージとして届く**（一斉送信と同じ思想）
- 参加/不参加の返答を受け取れる

---

### 10. タブ構成

```
🏠 ホーム / 🔍 発見 / ＋ 投稿 / 🔔 通知 / 💬 トーク
```

---

### 11. Circle組閣ウィザード（index.html）

初回セットアップ専用の画面。

- ステップバイステップで上位Circle（Inner）から順にフォロワーを選んでいく
- 全員を分類する必要はない。上位層だけ選べば、残りは自動でAmbientになる

## 非公開情報まとめ

Redianceでは以下の情報が本人以外に見えない：

| 情報 | 誰が見れるか |
|------|-------------|
| Circle階層の内容 | 本人のみ |
| つながり（コネクト）数 | 本人のみ |
| フォロワー数 | 本人のみ |
| カスタムグループ | 本人のみ |
| 一斉送信であること | 送り手のみ |
| アーカイブ | 本人のみ |

## 技術メモ

| 項目 | 内容 |
|------|------|
| 現状のモック | HTML + CSS + JS、1ファイル完結、フレームワークなし |
| 実装想定 | React Native / Flutter 等のモバイルフレームワーク |
| 影響力スコア計算 | バックエンド側でPageRank的アルゴリズムを実装 |
| AIデザイン生成 | 画像生成AI（Stable Diffusion等）との連携を想定 |
| Circle管理・スコア更新 | リアルタイムではなくバッチ処理でOK（称号がリアルタイムで変動するとUXが悪い） |

## 将来ビジョン

### 1. 影響力に経済力を統合
- 金融データ連携（Plaid / Moneytree等）でクレカ・銀行明細を取得
- ユーザーが自発的に連携 → 経済力スコアに反映
- 影響力 = 社会的信頼 + コンテンツ力 + 経済力の3層構造

### 2. Rediance決済
- Stripe組み込みでQR決済・クレカ・Apple Pay対応
- 加盟店での購買データを自動記録

### 3. 店舗連携（ビーコン）
- Apple iBeaconで入店を自動検知
- 店員のタブレットに顧客のカード（称号・経済力）が即表示
- 高級店が「VIP判定を自動化できる」メリットで導入を狙う
