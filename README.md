# Rediance — UIモック仕様書

SNS「Rediance」のUIプロトタイプです。プログラマがモックを見ながら仕様を理解するための資料です。

---

## プロジェクト概要

**コンセプト：「フォロワーの数だけが影響力じゃない」**

Redianceは、フォロワーをただカウントするのではなく、**誰にどう評価されているか**で影響力を測るSNSです。

コネクト済みの相手を3段階の階層（Circle）に分類し、上位Circleに入れてもらうほど影響力スコアが上がります。計算は**加点のみ・√ダンピング**のシンプルな構造です：

```
RS = Σ(√(相手RS) × 階層係数) + Insights加点
```

影響力の高い人にCoreに入れてもらうほど、自分のスコアも大きく上がります。

**数字を見せない思想：** つながり数・フォロワー数は非公開。見えるのは影響力スコアとティアだけ。数ではなく質で勝負するSNS。

## つながりの仕組み

Redianceには3つの関係レイヤーがあります。

### フォロー（一方的）

- 誰でも自由にフォローできる
- フォローした相手の投稿がフィードに流れてくる
- 影響力計算にも一方的に貢献する（Follow係数 ×2）
- **メッセージは送れない**
- **Circle階層には入れられない**（Circle配置はコネクト済みの相手のみ）

### コネクト（双方向）

コネクトが成立するとメッセージが送れるようになる。

**コネクトの成立方法：**

| 方法 | フロー |
|------|--------|
| QRコード | 対面でスキャン → 即コネクト成立 |
| ID検索 | ハンドルネームで検索 → 即コネクト成立 |
| 電話帳同期 | アドレス帳から知り合いを検出 → 即コネクト成立 |
| SNS上で発見 | 下記フロー参照 |

**SNS上で発見した場合のフロー：**
1. Aさんが Bさんを「フォロー」する
2. Bさんに通知「Aさんがフォローしました」
3. Bさんが Aさんに「コネクト申請」を送る
4. Aさんに通知「Bさんからコネクト申請が届きました」
5. Aさんが承認 → コネクト成立
6. **自動的に相互フォロー状態になる**
7. メッセージが送れるようになる
8. コネクト済み一覧に**NEWバッジ**（7日間）が表示される

**コネクトしなかった場合：**
- フォロー関係は残る（Aさんは Bさんの投稿をフィードで見れる）
- Circle階層には入れられない（コネクト済みでないため）
- メッセージは送れない

### Circle階層（個人の裏設定）

- コネクト済みの相手を3段階に分類する**完全に個人的な設定**
- **相手にはどの階層に入れられているか見えない（通知もしない）**
- フォローだけの関係ではCircle階層に入れられない

## 画面一覧と仕様

### 1. ホーム画面

ホームは「自分中心」の画面。以下の要素が上から順に並ぶ：

1. **ストーリーズアイコン列**（横スクロール） — 24時間以内に投稿した人のアバターを表示。タップで投稿を全画面再生
2. **自分のポケモンカード** — ティア・RS・4指標
3. **アクションボタン** — 💬メッセージ送信 / 🎉イベント作成
4. **Circle階層メンバー一覧** — Core〜Galaxyまで全3階層。アバタータップでストーリーズ再生。多い階層は「もっと見る」で遷移
5. **Circle設定リンク** / **グループ管理リンク**

**ヘッダー：** [＋]投稿ボタン（左）／ Rediance（中央）／ [♡]通知（右）

---

### 2. ストーリーズ（全画面表示）

- フルスクリーン表示。インスタストーリーズと同じUX
- 上部にプログレスバー（複数投稿の進捗）
- 自動送り（5秒）。左タップで戻る、右タップで次へ
- 右上に✕ボタンで閉じる
- **左上のアバター/名前をタップ → 相手のプロフィールに遷移**
- コメント欄はなし。タグ付けされた人だけ「返事ストーリーズ」を作れる
- 投稿はAIが自動生成した背景デザイン付き（モックではCSSグラデーション）

**「✨ 強い光が届きました」：** フォローしていなくても、上位ティア（DiamondStar以上）ユーザーの投稿がストーリーズに流れてくる。ゴールド背景で特別表示。

---

### 3. 投稿フロー — AIデザイン提案

1. テキストを入力
2. **「✨ アイデンティティに残す」チェック（デフォルトOFF）**
   - ONの場合 → カテゴリ選択（既存 or 新規作成）→ プロフィールのアイデンティティ欄にも掲載される
   - OFFでも投稿は永久保存される（投稿タブに並ぶ）
3. 「AIデザインを生成」ボタンを押す
4. AIが4パターンの背景を提案（サイバー／ナチュラル／エレガント／ポップ）
5. ユーザーが好みのデザインを選んで投稿

---

### 4. 投稿のライフサイクル

**全投稿は永久保存。** 24時間で消えるストーリーズ概念はない。

```
投稿 → 永久保存（投稿タブに時系列で並ぶ）
         ＋
       アイデンティティONなら → カテゴリ別にプロフィールに掲載
```

- **投稿タブ** — 全投稿が時系列で永久に並ぶ
- **アイデンティティ** — 投稿時にカテゴリを選んで掲載。プロフィールに永久表示
- **ストーリーズアイコン列** — ホーム上部に「24時間以内に投稿した人」のアバターを横並び表示。タップで全画面再生。あくまで最近の投稿者を知らせる**通知窓**であり、投稿自体は消えない

---

### 5. プロフィール — ポケモンカード風 + アイデンティティ

プロフィール画面の構成：
1. **ポケモンカード** — ティア・RS・4指標（Trust/Prestige/Insights/The Stature）
2. **フォローボタン**（他人のプロフィールの場合）
3. **アイデンティティ** — 本人が残した投稿がカテゴリ別に並ぶ

#### ポケモンカード

ティアはLoL風の9段階制。RS（Radiance Score）の閾値で自動昇格／降格する：

| ティア | 閾値 | 種別 |
|------|------|------|
| 🌟 SuperNova | 上位10人 | 人数制限 |
| ⭐ Nova | 上位100人 | 人数制限 |
| 💎 DiamondStar | 500,000+ | 閾値 |
| 💚 EmeraldStar | 150,000+ | 閾値 |
| 🟦 PlatinumStar | 50,000+ | 閾値 |
| 🟨 GoldStar | 20,000+ | 閾値 |
| ⬜ SilverStar | 5,000+ | 閾値 |
| 🟫 BronzeStar | 1,000+ | 閾値 |
| ⬛ IronStar | 0-999 | 閾値 |

- ディビジョン（IV/III/II/I）やLP制度はなし。閾値を跨げば即昇格／降格
- ティア名の横に現在RSを `PlatinumStar (50.9K)` の形で併記
- 閾値は初期値。実データで後日チューニング

**重要：** スコアはフォロワー数ではなく計算された影響力値。1人フォロワーが減っても大きく変動しない。

#### アイデンティティ

「自分はこういう人間です」を投稿で語る、自己紹介を超えた自己表現の場。

- 本人がカテゴリを自由に作成（例：「🇸🇬 シンガポール」「💼 経営」「🤖 AI・テック」）
- カテゴリは**最終更新順に自動ソート**される。新しい投稿があったカテゴリが上に上がる
- 訪問者は上から見れば「最近この人が力入れてるテーマ」がわかる
- カテゴリ内はインスタグリッド風（3列、背景グラデーションがサムネイル）
- タグ付けされた人のプロフィールにも飛べる → 人→投稿→人→投稿…と深掘りが止まらない
- **アイデンティティ未設定のユーザー：** カテゴリを1つも作っていない場合、全投稿が自動的にアイデンティティとして掲載される。カテゴリを1つでも作り始めたら、通常のカテゴリ表示に切り替わる

### 6. Circle（コネクト階層）— 3段階（ダンバー数準拠）

| Circle | 上限 | 意味 | ダンバーの対応層 |
|--------|------|------|------|
| 🔥 Core | 最大10人 | あなたの核・最も信頼する存在 | 最親密（Support clique） |
| ⭐ Halo | 最大50人 | 光輪・親しい人 | 友達グループ（Band） |
| 🌌 Galaxy | 最大150人 | 銀河・顔が見える人 | 知り合い（Community） |

**Circle配置はコネクト済みの相手のみ。** フォローだけの関係ではCircle階層に入れられない。

**Connect上限：500人**（ダンバーの「顔見知り層」の上限）。上限に達したら「枠いっぱい」と断れる。内訳はCircle配置210人（Core 10 + Halo 50 + Galaxy 150）＋ 未配置Connect 290人。

**フォロー（片思い）は無制限。** Connect前の「見たい」意思表示は誰でも何人でも自由。

**階層は完全非公開。** 相手にはどの階層に入れられているか見えない。通知もしない。

**NEWバッジ：** コネクト成立後7日間、相手のアバター横にNEWバッジが表示される。

### 影響力（Radiance Score, RS）の計算モデル

**基本式：加点のみ・√ダンピング**

```
RS = 流入合計 + Insights加点

流入合計 = Σ(√(相手RS) × 階層係数)
Insights = max(0, 平均視聴率 - 20%) × 流入合計
```

誰か（Xさん）があなたと関係を持つと、Xさんの `√RS` に係数をかけた値が流れ込む。√で相手RSの青天井インフレを抑制している。

| 関係 | 1人あたり流入量 |
|---|---|
| 🔥 XのCore（10枠） | √(X.RS) × 40 |
| ⭐ XのHalo（50枠） | √(X.RS) × 10 |
| 🌌 XのGalaxy（150枠） | √(X.RS) × 7 |
| 🤝 Xがコネクト（Circle未配置） | √(X.RS) × 5 |
| 👁 Xが片思いフォロー | √(X.RS) × 2 |

**減点は無し。** Disconnectされても減点ではなく単に流入が止まるだけ。

**Insights（コンテンツ加点）：** 自分の投稿の平均視聴率が20%を超えた分だけ、流入合計に上乗せ。20%は視聴率の基準ライン。

### プロフィールの4指標

ポケカには影響力（総RS）に加え、内訳として4つの指標を表示する：

| 指標 | 内訳 | 意味 |
|---|---|---|
| 💖 **Trust**（信頼度） | Core + Halo + Galaxy 流入 | 深い信頼で積んだ影響力 |
| 💎 **Prestige**（ブランド力） | Connect + Follow 流入 | 広い認知で積んだ影響力 |
| 📝 **Insights**（コンテンツ力） | max(0, 視聴率-20%) × 流入合計 | 投稿の質で積んだ加点 |
| 🏛️ **The Stature**（柱） | 支援者（Circle/Connect/Follow元）のRS総和 | あなたを支える柱の太さ（独立指標） |

- **RS = Trust + Prestige + Insights**
- The Statureは独立指標。「誰に支えられてるか」のスケールを示す
- 各指標のラベルをタップすると説明モーダルが開く

### Stature支援者リスト

The Statureをタップすると、自分を支えている人（Circle/Connect/Follow元）の一覧画面が開く。

- **全ユーザーが閲覧可能**（他人のプロフィールからも見れる）
- ティア降順でソート（DiamondStar → EmeraldStar → ...）、同ティア内はRS降順
- 各行：アバター＋名前＋ティアバッジ＋RS
- **関係種別（Core/Halo/Galaxy/Connect/Follow）は表示しない**（Circle配置のプライバシーを守るため）
- 無限スクロール（ページネーションで負荷軽減）

### Connect配置のプライバシー

- 相手が自分をCore/Halo/Galaxyに入れたことは**本人に通知しない**
- Circle配置は片方向の私的な関係性評価
- Connectは双方向の明示関係なので通知してOK

**大将の例（検算）：** フォロワー680人、Base流入 39,138。視聴率50%（+30%分） → Insights +11,741。合計 RS ≈ **50,879** → 🟦 **PlatinumStar**

### 8. フォロワー一覧

- **本人のみ閲覧可能**（他人のフォロワー数・リストは非公開）
- 自分をフォローしている人の一覧
- 各フォロワー横に「コネクト申請」ボタン
- 申請済みのユーザーは「申請中」表示
- コネクト済みのユーザーは「コネクト済み」表示
- マイページの設定メニューからアクセス

### 9. メッセージ機能

**一斉送信（Redianceの売り機能）：**
- Circle階層 or カスタムグループを宛先に選択
- 複数グループ/階層を同時に選択可能
- **受け取った人には個別DMとして届く。一斉送信だとは一切わからない**
- 送り手だけが全員の反応を俯瞰できる「ダッシュボード画面」を見れる

**一斉送信ダッシュボード（送り手のみ）：**
- 自分の送信メッセージが表示
- 各人の返信が時系列で並ぶ（アバター＋名前付き）
- 閲覧専用 — ここからは返信できない
- 個別に返信したい場合は、相手の名前をタップして1:1トーク画面に遷移

**1:1トーク：**
- 通常のチャットUI
- コネクト済みのユーザー同士のみ

**カスタムグループ：**
- ユーザーが自由に作れる私的なグループ分け（例：「酒好き」「投資仲間」）
- 相手からはグループの存在が見えない
- メッセージ/イベント招待の宛先として使える

---

### 10. イベント作成

- イベント名・日時・場所を入力
- 宛先をCircle階層 or カスタムグループから選択
- **招待は個別メッセージとして届く**（一斉送信と同じ思想）
- 参加/不参加の返答を受け取れる
